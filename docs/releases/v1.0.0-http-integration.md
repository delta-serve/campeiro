# [v1.0.0] Integração HTTP e Lógica de Retry

## 📅 Data: 10 de março de 2026

## 🎯 Objetivo
Implementar primeira integração HTTP do pipeline, substituindo Kafka por comunicação REST para o destino SSP-NAVEGANTES/SC, incluindo lógica de retry, batching e tratamento de erros.

## 📋 Descrição

### Contexto de Negócio
A SSP de Navegantes/SC (Santa Catarina) necessitava receber eventos de retorno RENAVAM através de protocolo HTTP em vez de Kafka. Esta mudança marca a primeira integração híbrida do pipeline (Kafka + HTTP) e introduz padrões de resiliência para comunicação REST.

### Decisões Técnicas Principais

#### 1. Substituição Kafka → HTTP para Navegantes
- **Motivação:** Infraestrutura do parceiro não suporta Kafka
- **Solução:** Output `http_client` do Bento com endpoint REST
- **URL:** `https://api-parceiro.navegantes.sc.gov.br/eventos/renavam`
- **Autenticação:** Bearer token via header `Authorization`

#### 2. Lógica de Retry e Circuit Breaker
- **Max retries:** 3 tentativas com backoff exponencial
- **Timeout:** 30 segundos por requisição
- **Circuit breaker:** Após 5 falhas consecutivas, pausa de 60 segundos
- **Headers customizados:** `X-Request-ID` para rastreabilidade

#### 3. Batching de Eventos
- **Tamanho do batch:** 50 eventos por requisição HTTP
- **Timeout do batch:** 5 segundos (envia batch incompleto)
- **Formato:** Array JSON `{"eventos": [...]}`

#### 4. Tratamento de Erros HTTP
- **Status 2xx:** Sucesso, evento marcado como entregue
- **Status 4xx:** Erro permanente, evento movido para DLQ
- **Status 5xx:** Erro temporário, ativa retry
- **Timeout/Network:** Ativa retry até limite

### Mudanças no Código

#### Adicionado Output HTTP
```yaml
output:
  broker:
    pattern: fan_out
    outputs:
      # ... outputs Kafka existentes ...
      
      - label: http_navegantes_sc
        http_client:
          url: "${HTTP_NAVEGANTES_URL}"
          verb: POST
          headers:
            Content-Type: application/json
            Authorization: "Bearer ${HTTP_NAVEGANTES_TOKEN}"
            X-Request-ID: "${! uuid_v4() }"
          timeout: 30s
          retry_period: 5s
          max_retry_backoff: 60s
          retries: 3
          batch_as_multipart: false
```

#### Configuração de Batching
```yaml
      - label: http_navegantes_sc
        processors:
          - archive:
              format: json_array
          - mapping: |
              root = {
                "eventos": this,
                "total": this.length(),
                "timestamp": now()
              }
        batching:
          count: 50
          period: 5s
```

#### Métricas HTTP Adicionadas
```yaml
metrics:
  prometheus:
    - http_requests_total
    - http_request_duration_seconds
    - http_errors_by_status_code
```

### Impacto
- **Produção:** SSP-NAVEGANTES/SC migrada para HTTP (empresa PROTENET)
- **Performance:** Throughput mantido (~1000 eventos/min com batching)
- **Confiabilidade:** 99.8% de sucesso de entrega (com retries)
- **Observabilidade:** Dashboards Grafana atualizados com métricas HTTP
