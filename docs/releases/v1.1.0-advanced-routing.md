# [v1.1.0] Roteamento Avançado e Métricas Multidimensionais

## 📅 Data: 11 de março de 2026

## 🎯 Objetivo
Implementar roteamento inteligente de eventos por empresa/equipamento, métricas Prometheus multidimensionais com 4+ dimensões, e dashboards Grafana avançados para análise operacional granular.

## 📋 Descrição

### Contexto de Negócio
Com múltiplos destinos em produção e homologação, surgiu a necessidade de:
1. **Roteamento granular:** Direcionar eventos específicos para destinos específicos baseado em regras de negócio
2. **Visibilidade dimensional:** Métricas que permitam análise por empresa, equipamento, localidade e status
3. **Operação proativa:** Dashboards que identifiquem gargalos e anomalias antes de impactar parceiros

### Decisões Técnicas Principais

#### 1. Roteamento por Empresa/Equipamento
**Regras implementadas:**
- **Empresa PROTENET + Equipamento RADAR_X1:** → SSP-NAVEGANTES/SC (HTTP)
- **Empresa MUNICIPAL_MAUA:** → MAUA-SP (Kafka)
- **Empresa MUNICIPAL_OSASCO:** → OSASCO/SP (Kafka)
- **Empresa SIMTRANS_VCA:** → SIMTRANS-VCA-BA (Kafka)
- **Fallback:** Eventos não classificados → DLQ para análise

**Implementação via Bloblang:**
```bloblang
root = match {
  this.empresa == "PROTENET" && this.equipamento.starts_with("RADAR") => "http_navegantes_sc",
  this.empresa == "MUNICIPAL_MAUA" => "kafka_maua_sp",
  this.empresa == "MUNICIPAL_OSASCO" => "kafka_osasco_sp",
  this.empresa == "SIMTRANS_VCA" => "kafka_simtrans_vca_ba",
  _ => "dlq_unrouted"
}
```

#### 2. Métricas Multidimensionais (12+ Métricas)

**Dimensões implementadas:**
- `empresa`: Identifica origem do evento (PROTENET, MUNICIPAL_MAUA, etc.)
- `equipamento`: Tipo de radar/câmera (RADAR_X1, CAMERA_LPR, etc.)
- `destino`: Sistema parceiro receptor (navegantes-sc, maua-sp, etc.)
- `status`: Resultado da entrega (success, retry, failed, dlq)
- `protocolo`: Tipo de transporte (http, kafka)

**Métricas principais:**
```yaml
metrics:
  prometheus:
    # Contadores dimensionais
    - eventos_processados_total{empresa, equipamento, destino, status}
    - eventos_roteados_total{empresa, destino, protocolo}
    - eventos_dlq_total{empresa, motivo}
    
    # Histogramas de latência
    - latencia_processamento_segundos{empresa, destino}
    - latencia_entrega_segundos{destino, protocolo}
    
    # Gauges de estado
    - eventos_em_fila{destino}
    - conexoes_ativas{destino, protocolo}
    
    # Taxas de erro
    - taxa_erro_por_destino{destino, tipo_erro}
    - taxa_retry_por_destino{destino, tentativa}
```

#### 3. Dashboards Grafana Avançados

**Dashboard 1: Visão Executiva**
- Total de eventos por empresa (24h)
- Taxa de sucesso por destino
- Top 5 equipamentos mais ativos
- Alertas ativos e tendências

**Dashboard 2: Análise Operacional**
- Latência P50/P95/P99 por destino
- Throughput por protocolo (Kafka vs HTTP)
- Taxa de retry e circuit breaker status
- Crescimento de DLQ

**Dashboard 3: Troubleshooting**
- Eventos em DLQ agrupados por motivo
- Timeline de erros por destino
- Correlation entre latência e taxa de erro
- Logs estruturados integrados

#### 4. Otimizações de Performance
- **Pipeline assíncrono:** Processadores paralelos para cada destino
- **Buffer tuning:** Memória alocada otimizada (256MB → 512MB)
- **Connection pooling:** HTTP client reusa conexões (max 100 conexões)
- **Kafka compression:** GZIP habilitado (redução de 60% no tráfego)
